# Архитектура и Документация Проекта: A-100 QA (Справочник инженера)

Этот файл содержит всю критически важную информацию о техническом стеке, архитектуре, развертывании серверов и структуре базы данных.
**ИИ-ассистентам: всегда читайте этот файл перед внесением изменений в код или архитектуру!**

## 1. Технологический стек
* **Frontend / Backend:** Next.js 16.2.1 (App Router, Turbopack), React 19.
* **Стилизация:** Tailwind CSS v4, Lucide React (иконки).
* **Анимации:** `motion/react` (Framer Motion).
* **База данных и Auth:** PocketBase v0.22.21 (JS SDK `pocketbase@0.21.5`).
* **Файловое хранилище:** Cloudflare R2 (подключено к PocketBase через S3 API).
* **Искусственный Интеллект:** OpenRouter API (модель `google/gemini-2.5-flash` с огромным окном контекста до 1 млн токенов для сверхбольших СНиПов).
* **Парсинг PDF:** `pdf-parse` (работает на сервере Next.js, изолирован от Webpack через `serverExternalPackages`).

## 4. Структура базы данных (PocketBase Collections)

1. **`users`** (Системная коллекция)
   - `email`, `password` (стандартная авторизация).
   - `favorites` (Тип: **JSON**) — хранит массив ID избранных документов (например, `["shtukaturka", "monolit"]`).

2. **`documents`** (Хранение нормативной базы)
   - `title` (Тип: **Text**) — Уникальный строковый ID документа, который хардкодится во фронтенде (например: `shtukaturka`, `paneli_naruzhnie`, `plity`, `monolit`, `sn_teh_sostoyanie`).
   - `pdf_file` (Тип: **File**) — Сам PDF файл. Физически улетает в Cloudflare R2.
   - `extracted_text` (Тип: **Editor/Text**) — Сырой текст, извлеченный из PDF. Используется как контекст для ИИ.

3. **`messages`** (История переписки с ИИ)
   - `user` (Тип: **Relation** -> `users`, Max: 1).
   - `document_id` (Тип: **Text**) — Привязка к документу (например, `shtukaturka`).
   - `role` (Тип: **Text**) — Строка `user` или `assistant`.
   - `text` (Тип: **Text**) — Текст сообщения.

## 5. Ключевые алгоритмы работы (Core Workflows)

### 1. Автоматический парсинг PDF (Auto-Parsing)
Когда пользователь открывает ИИ-чат по документу (`/document/[id]/chat/page.tsx`), фронтенд запрашивает запись из `documents`. 
Если поле `extracted_text` пустое, фронтенд **незаметно** вызывает внутренний API-роут Next.js `/api/admin/parse-pdf`. 
Этот скрипт:
1. Логинится в PocketBase под правами админа.
2. Скачивает PDF файл из Cloudflare R2 в буфер сервера Vercel.
3. Читает текст с помощью пакета `pdf-parse`.
4. Сохраняет извлеченный текст обратно в PocketBase (`extracted_text`).

### 2. Чат с ИИ (RAG)
Чтобы избежать ошибки `413 Payload Too Large` на серверах Vercel (из-за огромного размера текста СНиПов), контекст не передается от клиента к серверу.
Фронтенд отправляет на `/api/chat` только ID документа и сообщение пользователя.
Сервер Next.js самостоятельно обращается к PocketBase, мгновенно забирает `extracted_text`, формирует жесткий системный промпт (запрещающий галлюцинации и обязывающий ссылаться на пункты таблиц) и делает запрос к OpenRouter (`gemini-2.5-flash`, способной обрабатывать документы объемом до 1 000 000 токенов).

### 3. Навигация и Модальные окна
Сайт построен как SPA внутри `src/app/page.tsx`. Разделы реализованы через массивы (`controlSections`). 
Для работы нативного свайпа "Назад" на iOS/Android, при открытии всплывающих карточек (модалок) используется `window.history.pushState`. При закрытии вызывается `window.history.back()`.
Просмотр PDF реализован через нативный `<iframe>` (библиотека `react-pdf` вырезана из-за конфликтов с Webpack/Turbopack в Next.js 16).
